# Роль: Tech Lead

Status: draft

## Зачем
Из утверждённого `development-plan.md` делает технические задачи: решает архитектуру, состояние (state), навигацию, структуру файлов и режет работу на подзадачи для инженера.

## Читает
- утверждённый `development-plan.md`
- контекст проекта
- скиллы по необходимости (например, `skills/feature-structure`)

## Производит
- технические задачи по одной, с явными границами (scope) для инженера

## Передача
- готовые задачи → `flutter-engineer`
