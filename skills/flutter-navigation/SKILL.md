# Навигация

Часть общего слоя в `common/navigation/`. UI и фичи зависят только от контракта `AppNavigator` — берут его из дерева (`context.read<AppNavigator>()`) и зовут переходы. Навигатор завязан на UI, поэтому раздаётся `Provider`-ом, а не кладётся в DI-контейнер. Конкретный роутер спрятан за абстракцией; его настройка отдельно — `references/routing.md`.

## Контракт

```dart
abstract class AppNavigator {
  void goNamed(String name, {Object? extra});

  Future<T?> pushNamed<T extends Object?>(String name, {Object? extra});

  void replaceNamed(String name, {Object? extra});

  void pop<T extends Object?>([T? result]);
}
```

Базовый набор; под приложение расширяют (например, path/query-параметры для deep links).

- **`goNamed`** — целевой экран становится текущим, предыдущий стек сбрасывается, назад нельзя. Верхний уровень: после логина → главный, смена раздела.
- **`pushNamed`** — кладёт экран поверх стека и ждёт результат (`await`), который вернёт его `pop`. Углубление: список → карточка.
- **`replaceNamed`** — заменяет текущий верхний экран целевым: текущий уходит, назад на него нельзя, остальной стек цел. Шаги без возврата: онбординг.
- **`pop`** — закрывает текущий экран, опционально возвращая результат тому, кто его `pushNamed`.

Данные передаёшь через `extra`. `name` — имя из таблицы маршрутов (заводится при настройке роутера).
