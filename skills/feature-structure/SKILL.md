# Скилл: структура фичи

Status: draft

## Когда применять

Используй при создании новой фичи или модуля во Flutter-приложении.

## Что делает

Помогает выбрать минимальную структуру папок и файлов, чтобы инженер не
выдумывал её заново.

## Перед созданием файлов

1. Определи имя фичи в `snake_case`.
2. Посмотри существующую структуру проекта.
3. Если в проекте уже есть похожая фича, повторяй её стиль.
4. Если структуры нет, используй минимальный каркас ниже.

## Минимальный каркас

```text
lib/
└── src/
    └── features/
        └── <feature_name>/
            ├── screens/
            ├── widgets/
            └── bloc/
```

## Ограничения

- Не добавляй `data/`, `domain/`, repository или DI, если задача этого не требует.
- Не подключай навигацию и состояние шире текущей задачи.
- Если нужен другой подход, верни вопрос `tech-lead` или пользователю.
