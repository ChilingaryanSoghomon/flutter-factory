# Общий слой приложения

Переиспользуемый код, инфраструктура приложения и источники за пределами приложения. Правило «одной фиче — локально, нескольким — в общее» относится к коду внутри границы приложения; сервер, хранение, платформа, устройство и внешние SDK живут в `common/` в своей зоне.

Уровень намеренно верхний. Папки добавляются по мере необходимости; какая-то часть разрастётся — вынесем её в отдельный скилл, пока всё описано здесь.

## Прикладные кирпичи

- **Виджеты (widgets)** — переиспользуемые UI-компоненты: без логики конкретной фичи, настройка снаружи через параметры.
- **Утилиты (functions)** — чистые помощники и расширения; без состояния и без выхода во внешний мир.
- **Модели (models)** — типы данных для нескольких фич и единый тип ошибки `AppFailure`.
- **Контракты (domain)** — общие интерфейсы интеграций; реализация прячется за контрактом.

## Инфраструктура

Сборка приложения и выход во внешний мир.

- **Запуск (core)** — точка входа, конфигурация, сборка приложения.
- **Навигация (navigation)** — переходы между экранами.
- **Хранение (storage)** — локальные данные приложения.
- **Платформа (platform)** — обёртки над нативными возможностями устройства: разрешения, состояние сети, информация об устройстве, системные сервисы. Наружу даём контракт, конкретный Flutter-плагин, SDK или platform channel прячем внутри реализации.
- **Сервер (server)** — выход к внешним серверам: группы запросов, серверные клиенты, DTO, маппинг ответов и перехватчики.
- **Логирование и аналитика (logging)** — события и ошибки.
- **Оформление (theme)** — цвета, типографика, темизация.

## Контейнер зависимостей (DI)

Механизм, который выдаёт зависимости по всему приложению. Лежит в `common/core/di/`. Код опирается на **абстракцию** — чтение (`resolve`) и регистрацию (`register`), — а не на конкретный контейнер, поэтому контейнер сменяем.

`diSetup` — единственное место, где реализации привязываются к контрактам. Нужен новый общий объект (репозиторий, сервис) — регистрируешь его в `diSetup` по тому же образцу. Достаёшь: в UI через `context.resolve<T>()`, вне UI — инъекцией через конструктор.

## Границы

Общий слой **не зависит от фич** — зависимость только в одну сторону. Не обобщаем раньше времени код внутри приложения: спорное сначала живёт в фиче, переносим сюда после реального переиспользования. Это не относится к источникам за пределами приложения: они сразу живут в своей зоне `common/`.
