# Каркас приложения

Одноразовый каркас: раскладывает стандартную структуру запуска и сборки зависимостей при создании нового приложения (или пересборке старого). Дальше на нём строят фичи; сам каркас почти не трогают.

Код — шаблоны в `references/`; раскладывает их агент и под конкретный апп адаптирует.

## Что закладывается

- **DI** → `common/core/di/`: абстракция доступа к зависимостям + контейнер + `diSetup`. Шаблон: `references/di.md`.
- **Запуск** → `main.dart` + `common/core/app/`: точка входа, ран-функция (инициализация до UI, перехват ошибок), оболочка `App`/`MaterialApp`, splash-гейт. Шаблон: `references/bootstrap.md`.

## Порядок

`main` → ран-функция (`ensureInitialized`, внешние сервисы, запуск в зоне перехвата ошибок) → `App` (оболочка, `MaterialApp`) → splash собирает DI → поддерево приложения уже с доступом к зависимостям через `context.resolve`.

Тема и настройки оболочки в каркас не входят — добавляются отдельно. Конкретный UI splash-а и состав фич — под приложение.
