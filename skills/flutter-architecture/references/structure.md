# Каноничное дерево папок

Целевая структура. Папки добавляются по мере необходимости — если слоя ещё нет, это нормально, пока границы соблюдаются.

```text
lib/
├── main.dart
├── main_dev.dart                      # optional
├── main_prod.dart                     # optional
└── src/
    ├── common/                        # общая зона: ядро и всё, что нужно 2+ фичам
    │   ├── core/                      # сборка и запуск: app, config (AppConfig + remote config), di
    │   ├── navigation/                # контракт навигации, router, guards, нав-виджеты
    │   ├── domain/                    # общие контракты и модели интеграций
    │   ├── functions/                 # утилиты
    │   ├── server/                    # внешний серверный слой приложения
    │   │   └── <source_name>/         # отдельный backend/provider/base URL
    │   │       ├── api/               # запросы и server clients источника
    │   │       ├── dto/               # request/response модели источника
    │   │       ├── mappers/           # перевод DTO во внутренние типы приложения
    │   │       ├── errors/            # перевод ошибок источника в ошибку приложения
    │   │       └── interceptors/      # optional: headers/auth/logging
    │   ├── logging/                   # контракты логирования/аналитики + реализации
    │   ├── models/                    # общие типы данных, AppFailure
    │   ├── platform/                  # разрешения, сеть, инфо об устройстве
    │   ├── storage/                   # локальное хранение (key-value и т.п.)
    │   ├── theme/                     # тема-система
    │   └── widgets/                   # общие UI-компоненты
    │
    └── features/                      # фичи: самостоятельные срезы
        └── <feature_name>/
            ├── <feature_name>.dart    # optional public barrel фичи
            ├── screens/               # экраны уровня роута
            ├── widgets/               # UI-части этой фичи
            ├── bloc/                  # логика событие → состояние
            ├── domain/                # контракты (если есть доступ к внешним данным)
            └── data/                  # реализации и внешние вызовы
```

Заметки:

- **Навигация** — под `common/` (часть общей зоны), а не отдельной верхней папкой: у нас две зоны — общее и фичи.
- **packages/** (вынос крупных интеграций в отдельные пакеты) — пока не используем, всё в `lib`. Появится, когда часть кода станет общей между приложениями.
- Серверный слой приложения живёт в `common/server/`. Каждый внешний backend/provider/base URL получает свою папку, даже если от него нужен один запрос.
- Внутри feature `domain/` и `data/` держим файлы прямо, пока их мало; подпапки (`entities/`, `usecases/`, `repositories/`, `datasources/`, `mappers/`) добавляем только когда они реально улучшают читаемость.
