# Конвенции кода

Единые правила для всего кода фабрики. Скиллы демонстрируют их в примерах; единственный источник правды — здесь. Открой перед написанием кода.

## Два слоя — контракт и реализация

- **Контракт** — абстрактный класс с обычным именем по сути объекта.
- **Реализация** — то же имя с суффиксом `Impl`.
- Имя — по сути объекта; конкретный пакет/SDK остаётся в импорте.
- Оба слоя публичные. Потребитель зависит только от контракта и берёт его из DI-контейнера.
- Абстракция даёт независимость от пакета: реализацию можно сменить, не трогая потребителей.
- Один контракт может переиспользоваться несколькими потребителями.
- Иногда поверх контракта заводят дополнительный объект с осмысленными методами под конкретные данные — по месту, не как общее правило.

## Конструкторы

Обязательная зависимость — именованный `required`-параметр, присваивается в init-list в `final`-поле:

```dart
class SomeClass {
  SomeClass({required SomeDependency dependency}) : _dependency = dependency;

  final SomeDependency _dependency;
}
```

- `required` в Dart — именованный параметр, поэтому в `{}`.
- Исключение — приватный именованный конструктор с публичным полем (вида `_(this.field)`): оставляем как есть.

## Связывание

- Потребитель зависит от контракта и получает его из DI-контейнера.
- В контейнер регистрируют под типом контракта, а объектом дают реализацию: тип — абстракция, значение — `Impl`. Подмена реализации (в том числе дублёром в тесте) не трогает потребителя.
- Объект без контракта регистрируют под его собственным типом.
- Реализацию создают и связывают один раз при сборке приложения.

## Нейминг

Имя показывает роль и слой объекта — без сокращений, полным словом. Тип конкретного пакета или SDK в имя не попадает (он остаётся в импорте).

- **Файлы и папки** — `snake_case` (`feature_name_bloc.dart`).
- **Типы** — `UpperCamelCase`.
- **Контракт и реализация** — по разделу «Два слоя»: контракт — имя по сути объекта, реализация — то же имя с суффиксом `Impl`.

Внутри фичи имя собирается из имени фичи и роли, роль — в конце:

- **BLoC / Cubit** — `<Фича>Bloc` / `<Фича>Cubit`.
- **События** — базовый тип `<Фича>Event`; конкретное событие — осмысленным именем.
- **Состояния** — базовый тип `<Фича>State`; конкретное состояние — осмысленным именем.
- **Экраны** — роль `Screen` в конце имени: `<Назначение>Screen`.
- **Виджеты** — роль элемента в конце имени, чтобы из имени было видно, что это (`...Button`, `...Card`, `...Tile`).
