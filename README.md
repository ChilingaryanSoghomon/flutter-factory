# flutter-factory

Агентский комплект (kit) для разработки Flutter-приложений.
Чистый markdown — читается любым агентом (Claude Code, Codex, Gemini).

> Важно: этот файл агент НЕ читает сам по себе. Автоматически читается только
> `AGENTS.md` в КОРНЕ проекта. Он и подключает этот комплект (см. ниже).

## Как подключить в проект

1. Склонируй комплект в проект: `.agents/flutter-factory/`.
2. В КОРНЕВОЙ `AGENTS.md` проекта добавь строку-указатель:
   > Для Flutter-работы сначала прочитай и выполняй `.agents/flutter-factory/README.md`.
3. Агент авточитает корневой `AGENTS.md` → видит указатель → сам открывает комплект.

## Как описаны скиллы

Источник правды — сам скилл, а не отдельный JSON.
Каждый скилл лежит в `skills/<name>/SKILL.md` и начинается с frontmatter:

```yaml
---
name: <kebab-case-имя>
description: <одна строка: когда применять этот скилл>
---
```

`description` — то, по чему агент решает, нужен ли скилл.
Список скиллов ниже — «витрина» для агентов, которые сами папку не сканируют.
JSON-индекс не ведём руками; при нужде его генерирует скрипт из frontmatter.

## Роли

- `roles/tech-lead.md` — из утверждённого плана делает технические задачи.
- `roles/flutter-engineer.md` — выполняет задачи по одной.

## Скиллы

- `skills/feature-structure/` — структура новой фичи/модуля.

## Шаблоны

- `templates/development-plan.md` — единый входной артефакт.
