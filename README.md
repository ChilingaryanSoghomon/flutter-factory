# flutter-factory

Комплект для работы с Flutter-приложениями.

Используй его, когда нужно создать, изменить, улучшить или исправить
Flutter-приложение.

У скиллов в начале файла есть `description`. По нему агент решает, когда
использовать файл. Ниже — каталог: короткий триггер на каждый скилл для
быстрого подбора; полный `description` — в самом файле.

## Правила

Единые правила кода — в `conventions.md` (нейминг, конструкторы, слои). Скиллы
их демонстрируют в примерах; единственный источник правды — этот файл.

## Процесс

Ведение задачи (проектирование, планирование, отладка) — на твоём процессе или на superpowers, если он установлен. flutter-factory отвечает за Flutter-часть:

- **Проектирование** (новая фича, крупное или неясное изменение): сначала план, потом код. С superpowers — `brainstorming` → `writing-plans`.
- **Реализация**: код по `conventions.md`; с superpowers план ведут `executing-plans` или `subagent-driven-development`, баг — `systematic-debugging`.
- Перед «готово»: `flutter analyze` и `flutter test`.

## Скиллы

Знание по теме. Лежат в `skills/<name>/SKILL.md`; агент подбирает по теме задачи.

- **flutter-architecture** — границы и зоны приложения в целом, направление зависимостей, изоляция фич.
- **flutter-common** — общий слой: что переиспользуют несколько фич; верхнеуровневая карта общей зоны.
- **flutter-feature** — внутреннее устройство одной фичи: слои, поток данных, границы.
- **flutter-app-scaffold** — создание нового приложения или пересборка ядра; одноразовый каркас запуска.
- **flutter-navigation** — переходы между экранами через контракт навигации, охранники переходов, нав-элементы.
- **flutter-server** — общение с внешними серверами: запросы, DTO, маппинг во внутренние типы, ошибки источника.
- **flutter-storage** — локальное хранение: настройки и флаги, секреты, структурированные данные в локальной БД.
- **flutter-theme** — оформление: тема, светлая и тёмная, токены, типографика, применение в UI.
- **flutter-observability** — видимость работы приложения: аналитика, логирование, отчёты о сбоях.
- **flutter-testing** — написать или поправить тесты: модульные, логики состояния, виджет-тесты.
